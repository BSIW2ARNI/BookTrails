## Отдельный раздел про NFC

README должен содержать отдельный технический раздел **«NFC-механизм»**.

Нужно описать его в двух частях:

### 1. Доменная NFC-логика на backend

Обязательно зафиксируй:

- NFC-метка идентифицируется через `uid`;
- `TagBind` хранит активную привязку метки к экземпляру;
- `bind-tag` связывает UID с конкретным экземпляром;
- `unbind-tag` завершает активную привязку;
- `scan` ищет активную привязку по `tag_uid`;
- после scan экземпляр автоматически назначается текущему пользователю;
- при scan экземпляр переводится в статус `with_reader`;
- scan создаёт запись `Move` и обновляет поле текущего держателя у `Copy`;
- карточка экземпляра после scan должна показывать новый статус и держателя.

### 2. NFC на mobile

Нужно описать только фактический текущий уровень реализации:

- в Dart-слое уже есть platform channel `booktrail/nfc`;
- есть методы `isAvailable` и `scanTag`;
- scan screen умеет либо считать UID через platform channel, либо использовать ручной fallback;
- Android native handler должен возвращать `{ "uid": "..." }`;
- в репозитории нужно описывать AndroidManifest-разрешения и ожидаемый контракт platform channel;
- если native Android handler не полностью реализован в текущем коде, это нужно прямо и спокойно зафиксировать как текущее ограничение, не превращая это в roadmap.

## Отдельный раздел про ML-рекомендации

README должен содержать отдельный раздел **«Рекомендательная модель»**.

Нужно подробно описать реальную реализацию из `web/recommendation_engine.py`.

Обязательно зафиксируй:

- что это content-based recommendation engine;
- что используется `scikit-learn`;
- что текстовое представление книги строится из названия, авторов, жанра, языка, описания и текстов отзывов;
- что используется `TfidfVectorizer` с `ngram_range=(1, 2)` и `max_features=5000`;
- что схожесть считается через `cosine_similarity`;
- что положительными для пользователя считаются книги с review rating >= 4;
- что профиль пользователя строится как взвешенная сумма TF-IDF-векторов позитивно оценённых книг;
- что уже просмотренные/оценённые книги исключаются из выдачи;
- что при отсутствии позитивных оценок используется fallback по популярности и рейтингу;
- что результаты сохраняются в таблицу `Recommendation`;
- что есть management command `python manage.py rebuild_recommendations`;
- что рекомендации используются и в web, и в mobile API.

Нельзя называть эту модель “AI-помощником”, “нейросетью”, “LLM” или чем-либо подобным. Нужно использовать точную терминологию текущей реализации.

## Установка и запуск

Опиши пошаговый локальный запуск с нуля.

Используй реальные команды проекта:

- `python -m venv .venv`
- Windows PowerShell: `.venv\\Scripts\\Activate.ps1`
- macOS/Linux: `source .venv/bin/activate`
- `pip install -r requirements.txt`
- `python manage.py migrate`
- `python manage.py seed_booktrail`
- `python manage.py runserver`
- `python manage.py createsuperuser`

Также укажи запуск mobile:

- `cd booktrail_mobile`
- `flutter pub get`
- для эмулятора: `flutter run --dart-define=API_BASE_URL=http://10.0.2.2:8000/api/v1`
- для физического Android-устройства по USB с reverse: `flutter run --dart-define=API_BASE_URL=http://127.0.0.1:8000/api/v1`

Нужно указать реальные адреса:

- web: `http://127.0.0.1:8000/`
- каталог: `http://127.0.0.1:8000/catalog/`
- login: `http://127.0.0.1:8000/auth/login/`
- profile: `http://127.0.0.1:8000/profile/`
- scan: `http://127.0.0.1:8000/scan/`
- Django Admin: `http://127.0.0.1:8000/admin/`
- mobile API base: `http://127.0.0.1:8000/api/v1` или `http://10.0.2.2:8000/api/v1`
